# PRD — Snap Teleprompter
**Product Requirements Document**
버전 1.0 · 2026-06-25

---

## 1. 제품 개요

### 1.1 한 줄 정의
맥북 노치를 활용한 발표용 텔레프롬프터. 카메라를 바라보며 대본을 읽고, 폰으로 원격 제어한다.

### 1.2 배경
- 기존 텔레프롬프터 앱(moody 등)은 $29 이상의 가격에 핵심 기능만 제공
- 맥북 노치는 항상 화면에 존재하지만 활용되지 않는 공간
- 발표자는 프레젠테이션 중 대본 컨트롤을 위해 자리를 이탈해야 하는 문제 존재
- AI 코딩 도구의 발전으로 소규모 팀도 빠른 제품 출시 가능

### 1.3 기회
- 맥북 노치 특화 → 경쟁 제품과 명확한 차별점
- 발표/유튜브/강의 크리에이터 시장의 성장
- Free → Pro 전환 구조로 입소문 기반 성장 가능

## 2. 타겟 사용자

### 페르소나 A — 발표자 (핵심)
- **직업**: 직장인, 스타트업 창업자, 강사
- **상황**: PPT/Keynote 발표 중 대본이 필요한 사람
- **문제**: 발표 중 노트북으로 시선이 가면 어색함. 리모컨이 없으면 멈추기 불편
- **예시**: jak_gae (오늘 발표인데 리모컨이 없어서 불편했다)

### 페르소나 B — 콘텐츠 크리에이터
- **직업**: 유튜버, 온라인 강의 제작자
- **상황**: 카메라 앞에서 대본을 읽으며 녹화
- **문제**: 화면에 대본이 보이면 눈이 카메라를 향하지 않음. 노치 영역 활용으로 해결

### 페르소나 C — 개발자 / 오픈소스 애호가
- **직업**: 개발자
- **상황**: 무료로 쓰다가 발표 직전 Pro 업그레이드
- **특성**: GitHub에서 발견, 소스 코드 신뢰 → 구매 전환율 높음

## 3. 문제 정의

| 문제 | 현재 해결책 | 불편함 |
|------|------------|--------|
| 발표 중 대본 읽기 | 별도 모니터, 종이 | 자연스럽지 않음 |
| 텔레프롬프터 조작 | 노트북 키보드 | 발표 중 이동 필요 |
| 고가 앱 | moody $29, Speeko $49/년 | 가격 부담 |
| 슬라이드 + 대본 동기화 | 수동으로 병행 | 두 가지를 동시에 |
| 노치 영역 낭비 | 없음 | 활용 불가 |

## 4. 비즈니스 모델

### 4.1 가격 정책
```
Free         — 영구 무료 (기본 텔레프롬프터)
Pro          — $7.99 원타임 구매
얼리버드     — 초기 200명 $4.99 (출시 후 2주)
```

### 4.2 판매 채널
- **1차**: GitHub (오픈소스, 무료) + Gumroad (Pro 라이선스 키)
- **2차**: Mac App Store (샌드박스 제약으로 일부 기능 제한 가능)

### 4.3 수익 시나리오

| 시나리오 | 월 구매자 | 월 수익 (Gumroad 95%) |
|--------|----------|----------------------|
| 보수적 | 30명 | $228 |
| 중간 | 100명 | $760 |
| 낙관적 | 300명 | $2,280 |

### 4.4 Free/Pro 경계 원칙
- **Free**: "혼자 책상에서 연습하는" 시나리오를 완전히 커버
- **Pro**: "실제 발표 현장"에서 필요한 기능 전체

이 경계를 지키면 Free 사용자가 발표 전날 Pro로 전환하는 패턴 발생.

### 5.1 Free 기능 (현재 + 추가)

| F8 | 스크롤로 위치 이동 | ✅ 완료 |

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 성능 | 60fps 스크롤 유지. CPU 5% 이하 (재생 중) |
| 호환성 | macOS 13.0 이상. 노치 맥북 (MBP 2021+, MBA M2+) |
| 빌드 | Xcode 없이 Command Line Tools만으로 빌드 가능 |
| 보안 | 라이선스 키 로컬 검증. 서버 의존성 최소화 |
| 오프라인 | 모든 핵심 기능 인터넷 없이 동작 |
| 폰 리모컨 | 같은 WiFi 네트워크 내 동작 (LAN only) |

## 9. 제약 사항 및 리스크

| 리스크 | 영향도 | 대응 |
|--------|--------|------|
| App Store 샌드박스 제한 | 중 | 직판 우선, App Store는 라이트 버전 |
| 글로벌 키보드 모니터링 권한 거부 | 중 | 권한 없을 때 graceful fallback |
| 폰과 Mac 다른 WiFi | 중 | 에러 메시지 + 핫스팟 안내 |
| 라이선스 키 크랙 | 낮 | 초기엔 허용, 규모 커지면 서버 검증 |
