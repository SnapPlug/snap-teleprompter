### F9. 줄바꿈 개선

#### 개요
대본이 의미 단위(문장 끝)에서 자연스럽게 나뉘어 발표자가 읽기 편하도록 한다.

#### 문제
현재 CoreText 기반 줄바꿈은 패널 너비에 맞춰 글자 수 기준으로 자름. 문장 중간에서 잘리면 발표자가 읽기 어려움.

예시 (현재):
```
저는 오늘 AI 자동화가 어떻게 업무를 바꿀
수 있는지에 대해 말씀드리겠습니다.
```
예시 (개선 후):
```
저는 오늘 AI 자동화가
어떻게 업무를 바꿀 수 있는지에 대해
말씀드리겠습니다.
```

#### 사용자 스토리
> 발표자로서, 대본이 문장의 의미 단위에서 나뉘어지길 원한다. 그래야 읽기 자연스럽고 발표 흐름이 끊기지 않는다.

#### 기능 명세

**F9-1. 1차 분리: 문단 단위**
- 빈 줄 또는 `\n\n`으로 구분된 문단은 반드시 새 블록으로 시작
- 현재 동작과 동일 (유지)

**F9-2. 2차 분리: 문장 단위**
- 문장 종결 부호(`.` `?` `!` `。` `？` `！`) 뒤에서 우선적으로 줄바꿈
- 단, 줄 길이가 패널 너비의 40% 미만이면 다음 문장과 합침 (너무 짧은 줄 방지)

**F9-3. 3차 분리: 너비 초과 시 기존 CoreText**
- 문장 분리 후에도 너비를 초과하면 CoreText 기준으로 추가 분리

**F9-4. 사용자 강제 줄바꿈 존중**
- 대본에 `\n` 단일 줄바꿈이 있으면 해당 위치에서 반드시 줄바꿈

#### 처리 순서
```
입력 텍스트
  → 문단 분리 (\n\n 기준)
    → 각 문단을 문장 분리 (., ?, ! 기준)
      → 각 문장을 CoreText로 너비 체크
        → 너비 초과 시 CoreText 추가 분리
          → scriptLines 배열 완성
```

#### 수용 기준 (Acceptance Criteria)
- [ ] `가나다. 라마바.` → `["가나다.", "라마바."]` 로 분리됨
- [ ] 40자 이내 문장은 가능하면 합쳐짐
- [ ] 사용자가 입력한 `\n`은 무조건 줄바꿈 적용
- [ ] 마크다운 파일 로드 시에도 동일하게 적용

#### 엣지 케이스
- 문장 부호 없는 긴 텍스트: CoreText 기본 분리 적용
- `3.14`, `Mr.` 등 숫자/약어의 마침표: 다음 글자가 소문자 또는 숫자면 문장 종결로 보지 않음
- 빈 문자열 입력: scriptLines = []

### F10. 예상 발표 시간

#### 개요
현재 WPM 설정과 대본 분량을 기준으로 예상 발표 시간을 실시간으로 계산해 표시한다.

#### 사용자 스토리
> 발표 준비 중에, 내가 입력한 대본이 현재 속도로 몇 분짜리 발표인지 즉시 알고 싶다.

#### 기능 명세

**F10-1. 표시 위치**
- 메인 창 하단 컨트롤 바, 속도 슬라이더 옆
- 형식: `예상 발표 시간: 약 3분 20초`
- 대본이 비어 있으면 숨김

**F10-2. 계산 공식**
```
단어 수 = scriptText를 공백/줄바꿈으로 분리한 토큰 수
예상 초 = (단어 수 / wordsPerMinute) × 60
```

**F10-3. 실시간 갱신 트리거**
- `scriptText` 변경 시 (300ms debounce)
- `wordsPerMinute` 슬라이더 변경 시 (즉시)

**F10-4. 표시 형식**
```
60초 미만       → "약 N초"
60초 이상       → "약 N분 M초"
3600초 이상     → "약 N시간 M분"
```

#### 수용 기준
- [ ] 100단어 대본 + 100 WPM → "약 1분 00초" 표시
- [ ] WPM 슬라이더 드래그 중 실시간 갱신
- [ ] 대본 비우면 예상 시간 영역 사라짐

### P1. 발표 타이머

#### 개요
재생 시작부터 경과 시간을 노치 오버레이 하단에 표시한다.

#### 사용자 스토리
> 발표자로서, 발표 시작 후 경과 시간을 노치에서 바로 보고 싶다. 별도 스톱워치 없이.

#### 기능 명세

**P1-1. 표시 위치 및 형식**
```
┌────────────────────────────────────┐
│ ×   총 2개월이 걸렸고요.           ●  │
│     정량적인 수치상...               │
│ 02:34                              │  ← 왼쪽 하단
└────────────────────────────────────┘
```
- 형식: `MM:SS` (1시간 이상 시 `HH:MM:SS`)
- 색상: 흰색 20% 투명도 (방해 최소화)

**P1-2. 타이머 동작**
| 액션 | 타이머 동작 |
|------|------------|
| 재생(▶) | 시작 또는 재개 |
| 일시정지(⏸) | 정지 (경과 시간 유지) |
| 정지(■) | 초기화 (00:00) |
| 노치 탭 | 일시정지/재개 (타이머 동일하게) |

**P1-3. 구현 방식**
- `TeleprompterViewModel`에 `elapsedSeconds: Int` 추가
- 기존 60fps 타이머(`displayLink`)에서 초 단위 카운트업
- 별도 Timer 불필요

#### 수용 기준
- [ ] 재생 시작 시 `00:00`부터 카운트
- [ ] 일시정지 시 숫자 멈춤, 재개 시 이어서 진행
- [ ] 정지 시 `00:00` 초기화
- [ ] Pro 잠금: 비Pro 사용자에게 타이머 영역 미표시

### P2. 배경색 선택

#### 개요
노치 오버레이 배경색을 검정(기본) 또는 흰색으로 선택할 수 있다.

#### 사용자 스토리
> 밝은 조명이 필요한 발표 환경에서, 노치 배경을 흰색으로 설정해 얼굴에 빛이 더 반사되도록 하고 싶다.

#### 기능 명세

**P2-1. UI**
- 메인 창 컨트롤 바에 토글 버튼 추가
- 아이콘: 달 모양(어두운 모드) ↔ 해 모양(밝은 모드)

**P2-2. 색상 연동**
| 설정 | 배경색 | 글자색 | 상태 표시(●) |
|------|--------|--------|------------|
| 어두움(기본) | 검정 `#000000` | 흰색 | 초록/흰색 |
| 밝음 | 흰색 `#FFFFFF` | 검정 | 초록/검정 |

**P2-3. 상태 저장**
- `UserDefaults`에 저장. 앱 재시작 시 유지.

#### 수용 기준
- [ ] 토글 즉시 반영 (노치 재시작 불필요)
- [ ] 앱 재시작 후에도 선택 유지
- [ ] 흰색 모드에서 × 버튼, 상태 점도 색상 전환

### P3. 노치 영역 크기 조절

#### 개요
노치 오버레이의 너비와 높이를 사용자가 슬라이더로 조절할 수 있다.

#### 사용자 스토리
> 더 많은 텍스트를 한번에 보고 싶어서, 노치 패널을 더 넓고 높게 조절하고 싶다.

#### 기능 명세

**P3-1. 조절 범위**
| 속성 | 최소 | 기본 | 최대 |
|------|------|------|------|
| 너비 | 280px | 340px | 600px |
| 높이 | 60px | 110px | 220px |

**P3-2. UI**
- 메인 창 Pro 설정 영역에 슬라이더 2개 추가
- 레이블: `패널 너비: 340px` / `패널 높이: 110px`
- 노치가 열려 있을 때 슬라이더 변경 → 즉시 윈도우 크기 갱신

**P3-3. 제약**
- 화면 너비보다 클 수 없음 (`min(설정값, screen.frame.width - 40)`)
- 크기 변경 시 텍스트 재랩핑 트리거 (`panelWidth` 변경 → `wrapScript` 재실행)

**P3-4. 상태 저장**
- `UserDefaults`에 저장. 앱 재시작 시 유지.

#### 수용 기준
- [ ] 노치 열린 상태에서 슬라이더 조작 시 즉시 크기 변경
- [ ] 너비 변경 시 텍스트 줄바꿈 재계산
- [ ] 최대값 초과 불가 (화면 너비 제한)

### P4. 폰 리모컨

#### 개요
Mac과 같은 WiFi에 연결된 스마트폰 브라우저에서 텔레프롬프터를 제어한다. 별도 앱 설치 불필요.

#### 사용자 스토리
> 발표 중에 스마트폰으로 텔레프롬프터를 제어하고 싶다. 앱 설치 없이, 브라우저에서 바로 쓰고 싶다.

#### 기능 명세

**P4-1. 연결 방식**
```
[Mac 앱]
  → NWListener 로 HTTP 서버 시작 (포트 8765)
  → 메인 창에 QR코드 표시 (URL: http://[맥IP]:8765)
  → 폰에서 URL 접속 → HTML 컨트롤 UI 서빙

[폰 브라우저]
  → WebSocket 연결 (ws://[맥IP]:8765/ws)
  → 버튼 탭 → WebSocket 메시지 전송 → Mac 앱 명령 실행
```

**P4-2. 컨트롤 UI (폰 브라우저)**

```
┌─────────────────────────────┐
│     Snap Teleprompter       │
│          리모컨              │
├─────────────────────────────┤
│                             │
│      ◀◀    ⏸    ▶▶        │
│    (역방향) (정지) (재생)    │
│                             │
├─────────────────────────────┤
│  속도                        │
│  [−10 WPM]   130 WPM  [+10 WPM] │
│                             │
├─────────────────────────────┤
│       [처음으로 ↩]           │
└─────────────────────────────┘
```

**P4-3. WebSocket 메시지 프로토콜**
```json
// 폰 → Mac
{ "action": "play" }
{ "action": "pause" }
{ "action": "stop" }
{ "action": "speed", "delta": 10 }
{ "action": "speed", "delta": -10 }
{ "action": "rewind" }
{ "action": "reverse" }  // 역방향 재생

// Mac → 폰 (상태 동기화)
{ "state": "playing", "wpm": 130, "elapsed": 142 }
{ "state": "paused", "wpm": 130, "elapsed": 142 }
```

**P4-4. QR코드 UI (Mac 앱 메인 창)**
```
┌──────────────────────────────────┐
│  📱 폰 리모컨                    │
│                                  │
│  [QR코드 이미지 160×160]         │
│                                  │
│  http://192.168.0.5:8765         │
│  같은 WiFi에서 접속하세요          │
│                                  │
│  ● 연결됨 (iPhone)               │  ← 연결 시 표시
└──────────────────────────────────┘
```

**P4-5. 오류 처리**
| 상황 | 처리 |
|------|------|
| 포트 8765 이미 사용 중 | 8766 시도 후 사용 가능한 포트 표시 |
| WiFi 없음 | "WiFi에 연결해주세요" 메시지 |
| 폰 연결 끊김 | 상태 표시 "● 연결 대기 중" 으로 변경, 자동 재연결 대기 |

**P4-6. 역방향 재생**
- `pixelsPerSecond`를 음수로 설정 (`verticalOffset` 증가)
- 첫 번째 줄 위로 가면 정지

#### 수용 기준
- [ ] QR코드 스캔 후 5초 내 연결
- [ ] 재생/일시정지 버튼 응답 지연 500ms 이하
- [ ] 속도 변경 폰에서 조작 시 Mac 슬라이더도 즉시 반영
- [ ] 폰 연결 없어도 Mac 앱 정상 동작
- [ ] 동시 접속 최대 1대 (2대 이상 접속 시 첫 번째 연결 우선)

### P5. 전역 단축키 강화

#### 개요
다른 앱이 포커스를 갖고 있을 때도 작동하는 단축키를 확장한다.

#### 현재 전역 단축키
| 키 | 동작 |
|----|------|
| `<` | 속도 −10 WPM |
| `>` | 속도 +10 WPM |

#### 추가 전역 단축키 (P5)
| 키 조합 | 동작 |
|---------|------|
| `Space` | 재생 / 일시정지 (전역) |
| `R` | 처음으로 되감기 |
| `[` | 속도 −10 WPM (< 대안) |
| `]` | 속도 +10 WPM (> 대안) |

#### 제약 및 주의
- `Input Monitoring` 권한 필요. 미허용 시 전역 단축키 비활성 + 안내 표시
- `Space`는 전역 충돌 가능성 높음 → 설정에서 ON/OFF 토글 제공

#### 수용 기준
- [ ] 다른 앱 포커스 상태에서 Space로 재생/일시정지
- [ ] Input Monitoring 미허용 시 앱 크래시 없이 graceful fallback
- [ ] 설정에서 전역 Space 단축키 비활성화 가능

### P6. PDF 발표 자료 연동

#### 개요
PDF 파일을 불러와 메인 창 왼쪽에 슬라이드 미리보기를 표시하고, 대본과 함께 관리한다.

#### 사용자 스토리
> 발표 준비 시, PDF 슬라이드와 대본을 한 화면에서 보며 내용을 확인하고 싶다.

#### 기능 명세

**P6-1. UI 레이아웃 (PDF 로드 후)**
```
┌─────────────────┬────────────────────────────┐
│  슬라이드 미리보기 │  대본 편집기                │
│                  │                            │
│  [슬라이드 1]    │  안녕하세요. 오늘 발표는...  │
│  ───────────     │                            │
│  [슬라이드 2]    │  [다음 슬라이드]             │
│  ───────────     │                            │
│  [슬라이드 3]    │  AI 자동화란 무엇인가...     │
│                  │                            │
└─────────────────┴────────────────────────────┘
```
- 왼쪽 패널 너비: 200px (고정)
- PDFKit의 `PDFThumbnailView` 사용

**P6-2. 파일 불러오기**
- "파일 열기" 버튼 → .pdf 확장자 추가
- 드래그 앤 드롭으로 PDF 또는 텍스트 파일 수용

**P6-3. 슬라이드 클릭 동작**
- 왼쪽 슬라이드 클릭 → 해당 슬라이드 하이라이트
- 대본 내 `[다음 슬라이드]` 마커와 연동 (P7)

#### 수용 기준
- [ ] PDF 드래그 앤 드롭으로 불러오기
- [ ] 50페이지 이상 PDF에서 썸네일 렌더링 지연 없음
- [ ] PDF 없는 상태에서도 기존 UI 정상 동작

### P7. 슬라이드 페이지 마커

#### 개요
대본 내 `[다음 슬라이드]` 텍스트를 인식해 해당 시점에 노치에 시각 표시를 한다.

#### 사용자 스토리
> 텔레프롬프터를 보면서 발표할 때, 슬라이드를 넘겨야 할 타이밍을 노치에서 바로 알고 싶다.

#### 기능 명세

**P7-1. 마커 문법**
```
[다음 슬라이드]        ← 기본 문법
[slide]               ← 영어 대안
[→]                   ← 심볼 대안
```
- 대소문자 무시. 앞뒤 공백 무시.
- 마커 줄 자체는 노치에 텍스트로 표시되지 않음 (invisible)

**P7-2. 노치 표시**
```
마커 줄이 노치 상단 통과 시:
┌──────────────────────────────────┐
│ ▶ 다음 슬라이드                  │  ← 오버레이, 2초간 표시
│ 일반 대본 텍스트가 흐릅니다...   │
└──────────────────────────────────┘
```
- 배경: 파란색 반투명 배너
- 표시 시간: 2초 후 페이드아웃
- 연속 마커: 이전 배너 즉시 제거 후 새 배너 표시

**P7-3. PDF 연동 (P6 완료 시)**
- 마커 통과 시 왼쪽 슬라이드 패널의 선택 슬라이드 자동 이동
- 슬라이드 번호 순서대로 (1번째 마커 → 2번째 슬라이드로)

#### 수용 기준
- [ ] `[다음 슬라이드]` 줄이 노치 상단 통과 시 배너 표시
- [ ] 마커 텍스트는 스크롤 텍스트에 미표시
- [ ] 배너 2초 후 자동 사라짐

### P8. 슬라이드 자동 넘김

#### 개요
마커 통과 시 PDF의 페이지를 자동으로 넘긴다. (P6 + P7 선행 필요)

#### 기능 명세

**P8-1. 자동 넘김 방식**
- P7 마커 통과 이벤트 발생 시 → `PDFView.goToNextPage()` 호출
- 마지막 페이지에서는 자동 넘김 없음

**P8-2. 설정**
- 메인 창에 "자동 슬라이드 넘김" 토글 (기본: OFF)
- ON 상태에서만 자동 넘김 동작

#### 수용 기준
- [ ] 마커 통과 시 PDF 다음 페이지로 이동
- [ ] 토글 OFF 시 자동 넘김 없음 (P7 배너는 유지)
- [ ] 마지막 페이지에서 넘김 없음, 에러 없음

## 라이선스 잠금 명세

### 구조

```swift
// 기능 접근 시 공통 체크 패턴
func checkPro(feature: String, action: () -> Void) {
    if LicenseManager.shared.isPro {
        action()
    } else {
        showUpgradePrompt(feature: feature)
    }
}
```

### 라이선스 키 방식 (Gumroad 직판)

**구매 흐름:**
```
사용자 → Gumroad 결제
  → Gumroad가 라이선스 키 이메일 발송 (예: SNAP-XXXX-XXXX-XXXX)
    → 앱 내 "라이선스 키 입력" 창
      → 로컬 해시 검증
        → isPro = true, UserDefaults 저장
```

**검증 알고리즘:**
```
키 형식: SNAP-{A}-{B}-{C}
검증: SHA256(A + B + salt) 앞 4자리 == C
salt: 앱 번들 ID (com.snapavatar.snap-teleprompter)
```
- 서버 불필요. 오프라인 동작.
- 크랙 가능성 있으나 초기 단계에서는 허용 (규모 커지면 서버 검증 추가)

**업그레이드 프롬프트 UI:**
```
┌─────────────────────────────────────┐
│  🔒 Pro 기능                        │
│                                     │
│  발표 타이머는 Pro 기능입니다.         │
│                                     │
│  [라이선스 키 입력]  [Gumroad에서 구매] │
│                                     │
│  이미 키가 있으신가요?               │
└─────────────────────────────────────┘
```
