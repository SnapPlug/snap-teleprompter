# Snap Teleprompter — Free 기능 테스트 시나리오

> 테스트 환경: 노치가 있는 맥북 (MacBook Pro 2021 이후 / MacBook Air M2 이후), macOS 13.0 이상

---

## 사전 준비

1. `bash build.sh` 로 앱 빌드
2. `open SnapTeleprompter.app` 으로 실행
3. 처음 실행 시 Gatekeeper 경고 → Finder에서 우클릭 → 열기 → 열기
4. 테스트용 샘플 파일 준비:
   - `sample.txt` — 10문장 이상의 일반 텍스트
   - `sample.md` — `# 제목`, `**굵게**`, 리스트 포함된 Markdown

---

## T01 — 대본 직접 입력

**목적**: 텍스트 편집기에 직접 타이핑한 대본이 저장되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 앱 실행 후 중앙 편집기 영역 클릭 | 커서가 편집기에 위치 |
| 2 | "안녕하세요. 반갑습니다. 오늘 발표를 시작하겠습니다." 입력 | 입력한 텍스트가 그대로 표시 |
| 3 | 편집기가 비어있을 때 플레이스홀더 텍스트 확인 | "대본을 여기에 입력하거나 .txt / .md 파일을 불러오세요…" 표시 |
| 4 | 한 글자라도 입력하면 플레이스홀더 사라지는지 확인 | 플레이스홀더 즉시 사라짐 |
| 5 | 앱 재시작 후 편집기 내용 확인 | 빈 상태 (입력 내용은 세션 중에만 유지) |

## T02 — .txt 파일 불러오기

**목적**: 텍스트 파일을 열어 편집기에 내용이 로드되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 상단 **파일 열기** 버튼 클릭 | 파일 선택 패널 열림 |
| 2 | `sample.txt` 선택 후 열기 | 편집기에 파일 내용 로드 |
| 3 | 원본 파일과 편집기 내용 비교 | 내용 동일, 줄바꿈 보존 |
| 4 | 파일 로드 후 편집기에서 직접 수정 가능한지 확인 | 추가 타이핑 정상 작동 |

---

## T03 — .md 파일 불러오기 (Markdown 자동 제거)

**목적**: Markdown 파일 로드 시 서식 기호가 제거되고 텍스트만 남는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | **파일 열기** 클릭 → `sample.md` 선택 | 파일 로드 |
| 2 | `# 제목` → `제목` 으로 변환됐는지 확인 | `#` 기호 없음 |
| 3 | `**굵게**` → `굵게` 로 변환됐는지 확인 | `**` 기호 없음 |
| 4 | `- 리스트` → `리스트` 로 변환됐는지 확인 | `- ` 기호 없음 |
| 5 | `` `코드` `` → `코드` 로 변환됐는지 확인 | 백틱 없음 |
| 6 | `[링크](url)` → `링크` 로 변환됐는지 확인 | URL 제거, 링크 텍스트만 남음 |

## T04 — 노치 오버레이 표시 / 닫기

**목적**: 노치 패널이 정확한 위치에 열리고 닫히는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 편집기에 텍스트 입력 후 **노치에 표시** 버튼 클릭 | 화면 상단 노치 아래에 검정 패널 등장 |
| 2 | 패널 위치 확인 | 노치(카메라 범프)를 중심으로 좌우 대칭 배치 |
| 3 | 패널이 메뉴바 위쪽 공간을 덮지 않는지 확인 | 메뉴바 아이콘(시계, 배터리 등)이 가려지지 않음 |
| 4 | 다른 앱(Finder 등) 클릭 후 노치 패널 확인 | 패널이 항상 최상단에 유지됨 |
| 5 | **노치 닫기** 버튼 클릭 | 패널 즉시 사라짐 |
| 6 | 대본 없이 **노치에 표시** 클릭 | 패널 열림, "▶ 시작을 누르면 여기에 대본이 흐릅니다" 안내 텍스트 표시 |

---

## T05 — 재생

**목적**: 재생 버튼으로 대본 스크롤이 시작되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 대본 입력 → 노치 열기 → ▶ 재생 버튼 클릭 | 노치 패널에서 텍스트가 위에서 아래로 스크롤 시작 |
| 2 | 재생 중 노치 패널 우상단 상태 아이콘 확인 | 초록 점(●) 표시 |
| 3 | 메인 창 재생 버튼 아이콘 확인 | ▶ → ⏸ 로 변경 |
| 4 | 대본이 끝까지 스크롤되면 자동 정지 확인 | 스크롤 멈춤, 버튼 ▶ 로 복귀 |

## T06 — 일시정지 / 재개

**목적**: 일시정지 시 스크롤이 멈추고 정확한 위치에서 재개되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 재생 중 ⏸ 버튼 클릭 | 스크롤 즉시 정지, 위치 유지 |
| 2 | 일시정지 상태에서 노치 패널 우상단 확인 | ⏸ 아이콘 표시 |
| 3 | ▶ 버튼 클릭 | 멈춘 위치에서 스크롤 재개 |
| 4 | 5초 일시정지 후 재개 → 이전 위치와 동일한지 확인 | 위치 변화 없이 재개 |

## T07 — 정지

**목적**: 정지 시 스크롤이 초기화되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 재생 중 ■ 정지 버튼 클릭 | 스크롤 멈추고 처음 위치로 리셋 |
| 2 | 노치 패널 상태 확인 | 안내 텍스트 ("▶ 시작을 누르면…") 다시 표시 |
| 3 | 정지 후 ▶ 재생 클릭 | 대본 처음부터 다시 시작 |

## T08 — Space 키로 재생 / 일시정지

**목적**: Space 키가 재생과 일시정지를 토글하는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 노치 열린 상태에서 `Space` 키 누름 | 재생 시작 |
| 2 | 재생 중 `Space` 키 누름 | 일시정지 |
| 3 | 일시정지 중 `Space` 키 누름 | 재개 |
| 4 | 대본이 비어있을 때 `Space` 키 누름 | 아무 반응 없음 (비활성) |

## T09 — `>` / `<` 키로 속도 조절 (앱 내)

**목적**: 앱이 포커스된 상태에서 단축키로 속도가 변경되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 앱 클릭 후 `>` 키 누름 | 속도 슬라이더 10 WPM 증가 |
| 2 | `>` 키 반복 → 280 WPM 도달 후 추가 누름 | 280 WPM 유지 (상한 고정) |
| 3 | `<` 키 누름 | 속도 슬라이더 10 WPM 감소 |
| 4 | `<` 키 반복 → 20 WPM 도달 후 추가 누름 | 20 WPM 유지 (하한 고정) |

---

## T10 — `>` / `<` 키로 속도 조절 (전역, 다른 앱 포커스)

**목적**: 다른 앱이 활성화된 상태에서도 단축키가 작동하는지 확인

> **사전 조건**: 시스템 환경설정 → 개인 정보 보호 및 보안 → 입력 모니터링에서 SnapTeleprompter 허용

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | Snap Teleprompter 노치 열기, 재생 시작 | 스크롤 진행 중 |
| 2 | Finder 또는 다른 앱 클릭하여 포커스 이동 | 노치 스크롤 계속 진행 |
| 3 | 다른 앱이 활성화된 상태에서 `>` 키 누름 | 속도 10 WPM 증가 |
| 4 | `<` 키 누름 | 속도 10 WPM 감소 |
| 5 | 입력 모니터링 미허용 상태에서 동일 테스트 | 전역 단축키 작동 안 함 (정상 — 권한 없으면 무시) |

## T11 — 속도 슬라이더 조절

**목적**: 슬라이더로 속도를 변경하면 스크롤 속도가 즉시 반영되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 재생 중 속도 슬라이더를 오른쪽으로 드래그 | 텍스트 스크롤 속도 빠르게 변함 |
| 2 | 슬라이더를 왼쪽으로 드래그 | 스크롤 느려짐 |
| 3 | 슬라이더 값이 "XXX WPM" 형태로 표시되는지 확인 | 숫자가 실시간 업데이트 |
| 4 | 최솟값(20 WPM)에서 읽기 편한 속도인지 확인 | 매우 느리지만 스크롤됨 |
| 5 | 최댓값(280 WPM)에서 정상 작동 확인 | 빠르게 스크롤됨 |

| 5 | 22pt 최댓값에서 텍스트 잘림 여부 확인 | 노치 높이 안에서 정상 표시 |

## T13 — 노치 패널 탭으로 일시정지 / 재개

**목적**: 노치 패널을 탭(클릭)했을 때 재생이 토글되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 재생 중 노치 패널 텍스트 영역 클릭 | 일시정지, ⏸ 아이콘 표시 |
| 2 | 다시 노치 패널 클릭 | 재개, 초록 점 표시 |
| 3 | 재생 전(대본 없음) 상태에서 노치 탭 | 아무 반응 없음 |
| 4 | × 버튼 클릭이 일시정지와 충돌하지 않는지 확인 | × 버튼은 앱 종료, 탭과 별개 동작 |

## T14 — 스크롤로 위치 이동

**목적**: 노치 패널 위에서 스크롤 시 위치 이동 및 자동 일시정지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 재생 중 노치 패널 위에서 트랙패드 스크롤 위로 | 자동 일시정지 + 텍스트 위로 이동 (빠른 감기) |
| 2 | 노치 패널 위에서 트랙패드 스크롤 아래로 | 텍스트 아래로 이동 (되감기) |
| 3 | 스크롤 후 노치 탭 | 현재 위치에서 재생 재개 |
| 4 | 스크롤로 대본 맨 위 이전으로 이동 | 빈 화면 (한계값 유지, 크래시 없음) |
| 5 | 스크롤로 대본 맨 아래 이후로 이동 | 빈 화면 (한계값 유지, 크래시 없음) |

## T15 — 문장 단위 줄바꿈

**목적**: 문장 중간이 아닌 문장 경계에서 줄바꿈이 일어나는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 아래 텍스트 입력 후 노치 열기: `"안녕하세요. 오늘 발표를 시작합니다. 잘 부탁드립니다."` | — |
| 2 | ▶ 재생 후 텍스트 줄바꿈 위치 관찰 | `.` 이후에서 줄바꿈 발생 |
| 3 | 한 줄이 너무 짧으면 다음 문장과 병합되는지 확인 | 짧은 문장끼리 한 줄로 합쳐짐 |
| 4 | 영어 문장으로도 테스트: `"Hello. This is a test. Thank you."` | `.` 기준으로 분리 |
| 5 | `?` `!` `。` `？` `！` 로 끝나는 문장 테스트 | 모든 종결 부호에서 줄바꿈 |

## T16 — 예상 발표 시간

**목적**: 대본 분량에 따른 예상 발표 시간이 정확하게 계산되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 편집기에 텍스트 입력 | 하단 컨트롤 바에 "예상 발표 시간: 약 N분 M초" 표시 |
| 2 | 편집기가 비어있을 때 확인 | 예상 시간 표시 없음 |
| 3 | 속도 슬라이더 변경 | 예상 시간 즉시 재계산 |
| 4 | 대본 분량 늘림 | 예상 시간 증가 |
| 5 | 약 130 WPM 기준 130단어 입력 | 약 1분으로 표시되는지 확인 |
| 6 | 60분 이상 분량 입력 | "약 X시간 Y분" 형태로 표시 |

## T17 — 발표 타이머

**목적**: 재생 중 경과 시간이 노치에 표시되고 정확하게 동작하는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 재생 시작 | 노치 좌하단에 `00:00` 부터 카운트 시작 |
| 2 | 30초 후 타이머 확인 | `00:30` 표시 (±1초 허용) |
| 3 | 일시정지 시 타이머 확인 | 타이머 멈춤 |
| 4 | 재개 시 타이머 확인 | 멈춘 시점부터 이어서 증가 |
| 5 | 정지(■) 후 타이머 확인 | 타이머 `00:00` 리셋, 화면에서 사라짐 |
| 6 | 1시간 이상 경과 시뮬레이션 | `1:00:00` 형태로 전환 (HH:MM:SS) |

## T18 — 배경색 선택 (검정 / 흰색)

**목적**: 배경색 토글이 노치 패널에 즉시 반영되고 설정이 유지되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 노치 열린 상태에서 하단 ○ 흰색 버튼 클릭 | 노치 배경이 흰색, 텍스트가 검정으로 전환 |
| 2 | ● 검정 버튼 클릭 | 노치 배경이 검정, 텍스트가 흰색으로 전환 |
| 3 | 전환 시 딜레이 없이 즉시 반영되는지 확인 | 즉각 색상 변경 |
| 4 | 흰색 배경 선택 후 앱 재시작 | 재시작 후에도 흰색 유지 (UserDefaults 저장) |
| 5 | 흰색 배경에서 그라데이션(페이드) 확인 | 상/하단 페이드가 흰색 배경에 맞게 표시 |
| 6 | 흰색 배경에서 × 버튼, 상태 아이콘 색상 확인 | 검정(어두운) 색으로 전환 |

## T19 — 노치 크기 조절 (너비 / 높이)

**목적**: 슬라이더로 노치 패널 크기를 변경하면 즉시 반영되는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 노치 열린 상태에서 패널 너비 슬라이더 오른쪽 드래그 | 노치 패널 너비 즉시 확장 |
| 2 | 너비 슬라이더 왼쪽 드래그 | 패널 너비 축소 |
| 3 | 패널 높이 슬라이더 아래 드래그 | 패널 높이 증가, 텍스트 영역 넓어짐 |
| 4 | 패널 중앙 정렬 확인 | 너비 변경 후에도 화면 가로 중앙 유지 |
| 5 | 너비 280px 최솟값 설정 후 앱 재시작 | 재시작 후 280px 유지 |
| 6 | 높이 220px 최댓값 설정 후 텍스트 표시 확인 | 메뉴바 아이콘과 겹치지 않음 |
| 7 | 크기 변경 중 재생 상태 유지 확인 | 크기 조절 중에도 스크롤 계속 진행 |

## T20 — 화면 공유 숨김 (Zoom / Teams)

**목적**: 화면 공유 시 노치 패널이 상대방에게 보이지 않는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 하단 "화면 공유 숨김" 토글 ON 상태 확인 (기본값) | 스위치 켜짐 |
| 2 | 노치 열기 → macOS 스크린샷 (⌘⇧3) 촬영 | 스크린샷에 노치 패널이 **캡처되지 않음** |
| 3 | macOS 스크린샷 (⌘⇧4) 노치 영역 드래그 캡처 | 노치 패널이 **캡처되지 않음** |
| 4 | QuickTime → 화면 기록 시작 → 노치 확인 | 녹화 파일에 노치가 **나타나지 않음** |
| 5 | "화면 공유 숨김" 토글 OFF | 노치가 스크린샷/녹화에 **정상 캡처됨** |
| 6 | 토글 OFF 상태에서 ⌘⇧3 | 노치 패널이 스크린샷에 포함됨 |
| 7 | Zoom 화면 공유 (가능한 경우) — 토글 ON | 공유 화면에 노치 미표시 |
| 8 | 앱 재시작 후 토글 상태 확인 | 마지막 설정값 유지 |

## T21 — 설정 영속성 (앱 재시작 후 유지)

**목적**: UserDefaults에 저장된 설정이 재시작 후에도 유지되는지 통합 확인

| 설정 항목 | 변경값 | 재시작 후 기대 결과 |
|-----------|--------|---------------------|
| 패널 너비 | 500px | 500px 유지 |
| 패널 높이 | 150px | 150px 유지 |
| 배경색 | 흰색 | 흰색 유지 |
| 화면 공유 숨김 | OFF | OFF 유지 |

**절차**:
1. 위 4가지 설정 모두 기본값과 다른 값으로 변경
2. 앱 종료 (`⌘Q` 또는 × 버튼)
3. 앱 재실행
4. 각 설정값 확인

## T22 — 엣지 케이스

**목적**: 비정상 입력이나 극단적 상황에서 크래시 없이 안정적으로 동작하는지 확인

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| 1 | 대본 없이 ▶ 재생 클릭 | 버튼 비활성화 또는 반응 없음 (크래시 없음) |
| 2 | 이모지 포함 대본 재생: `"안녕하세요 😊 반갑습니다 🎉"` | 이모지 포함하여 정상 스크롤 |
| 3 | 매우 긴 한 줄 (500자 이상) 입력 후 재생 | 자동 줄바꿈 후 스크롤 (크래시 없음) |
| 4 | 빠른 속도로 토글: 노치 열기/닫기 10회 반복 | 메모리 누수 없이 정상 동작 |
| 5 | 재생 중 대본 텍스트 편집 | 노치 패널 텍스트가 300ms 후 업데이트 |
| 6 | 외부 모니터 연결/해제 중 노치 패널 동작 | 패널 위치 재조정 또는 유지 (크래시 없음) |
